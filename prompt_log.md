**פרומפט (תמצית):** בקשה לקרוא את קובץ תיאור המטלה, את חוברת ההרצאה L08

**תוצאה:** קריאת כל 5 המסמכים הרלוונטיים (טקסט המטלה, L08 summary, Part-A/B/C,

## שלב 6 — Phase 4 בפועל: uv sync, בדיקת עשן, מודל גדול, ותיקון קריטי במפרט החומרה

**פרומפטים (תמצית):** "תריץ uv sync" → "כן תוריד את אולמה עכשיו" (בשיחה קודמת) →
אישור להוריד מודל גדול (`qwen2.5:72b`) תוך דרישה מפורשת "תוודא שהכל מעודכן ב-PRD
PLAN TODO ובגיטהאב והסקרינשוטס בקבצים המתאימים... תוודא באמת מה צריך לעשות
בריידמי כל פעם".

**תוצאה:**
1. `uv sync --extra dev` — 175 חבילות הותקנו, כולל `airllm==3.0.1`,
   `torch==2.12.1+cpu`, `transformers==5.12.1`.
2. התגלתה ותוקנה תקלת `UnicodeEncodeError` (נתיב עברי + קונסולת Windows).
3. בדיקת עשן: `phi3:mini` דרך Ollama — הצלחה מלאה, נמדדו TTFT/TPOT אמיתיים.
4. הורדת `qwen2.5:72b` (47GB) והרצתו — **נכשלה טעינה תוך 6.28 שניות**
   ("unable to allocate CPU buffer").
5. **תיקון קריטי**: תוך כדי בדיקת מצב הזיכרון בזמן הכישלון, התגלה שמדידת ה-RAM
   המקורית (76GB, מהתחלת השיחה) הייתה שגויה פי 10 — RAM אמיתי ~7.65GB בלבד.
   תוקן ב-`docs/PRD.md` (v1.02), `docs/PLAN.md` ADR-1, `docs/PRD_model_loader.md`,
   ו-`README.md`. הכישלון של qwen2.5:72b מוסבר עכשיו באופן מדויק (לא פרגמנטציה,
   פשוט אין מספיק RAM), וסיפור הניגוד Baseline-נכשל/AirLLM-מצליח התחזק.
6. כל שלב תועד ב-README.md ("יומן ניסויים") + קובץ ראיות ב-`results/` + commit
   ו-push נפרד לכל שלב.

## שלב 7 — השלמת Phase 4: baseline רשמי, Q2_K, רגישות, Roofline, מיזוג README

**פרומפט:** "כן, תמשיך לסגור את הפערים וכמובן לעדכן את הקבצים המתאימים PRD PLAN
TODO אם צריך ואת ה-README כמו שמצוין בקובץ וכמובן פוש" (לאחר שדוח ביניים זיהה
3 פערים: Model Roofline/benchmark suite מלא, הרחבה מקורית, ומיזוג README).

**תוצאה:**
1. **Baseline רשמי** דרך `ModelLoaderService` (Phi-3-medium FP32, לא Ollama) —
   קריסת segfault (exit 139) תוך 17.3 שניות, שוחזרה פעמיים. ראיה: `results/
   baseline_official_phi3_medium_oom_crash.json`.
2. **קוונטיזציה Q2_K** (רמה שנייה) — התגלה ותוקן באג אמיתי: כל רמות הקוונטיזציה
   השתמשו בטג Ollama זהה (התעלמות מ-`quant_level` בפועל). נוסף `quant_ollama_tags`
   ל-`config/setup.json` (ADR-7). תוצאה: Q2_K מהיר וקל יותר מ-Q4_0 אך חצה את
   "קו האדום" של הדיוק (פלט לא קוהרנטי/לא רלוונטי).
3. **בדיקת רגישות מצומצמת** (Q4_0 ב-16 מול 30 טוקנים) במקום רשת מלאה בת 18
   ריצות שהייתה נמשכת מעל 5 שעות (הוסבר ותועד כהחלטת היקף מודעת).
4. **Model Roofline** הופק מנתונים אמיתיים — תוקנו 2 באגים אמיתיים ב-
   `ReportService`: קליטת קובצי ראיה בפורמט single-object/סכמה שונה, ומלכודת
   `NaN or x` (ADR-8).
5. **מולאו** `notebooks/results_analysis.ipynb` בתוצאות אמיתיות ומסקנות מלאות.
6. **מוזג הדוח הטכני המלא ל-`README.md`** לפי ex05 §8: מפרט חומרה, כל 7
   הניסויים, ניתוח כלכלי, Model Roofline, טבלת מענה מרוכז ל-6 שאלות המחקר,
   סעיף הרחבה מקורית — במקום ה"שלד ממתין ל-Phase 4" הקודם.
7. **תוקנו 3 כשלים נוספים** שנחשפו בהרצה ראשונה בפועל של `pytest`/`ruff` (מעולם
   לא הורצו קודם): dataclass test drift, טסט RAM-sampling חסר יציבות (flaky),

   mocked.py` זיהם את `results/` האמיתי בגלל `project_root` fixture בלי `chdir`.
