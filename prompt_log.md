# יומן פרומפטים (Prompt Engineering Log)

תיעוד השימוש בסוכן AI (Claude Code, Anthropic) בבניית פרויקט מטלה 5, לפי דרישת
`software_submission_guidelines-V3.pdf` §8.3.

## שלב 1 — קריאה והבנת המטלה

**פרומפט (תמצית):** בקשה לקרוא את קובץ תיאור המטלה, את חוברת ההרצאה L08
(LoRA/AirLLM), את כל החומרים בתיקיית Lecture 8, ואת קובץ הנחיות ההגשה
(`software_submission_guidelines-V3.pdf`), ולוודא הבנה מלאה לפני כניסה ל-Plan Mode.

**תוצאה:** קריאת כל 5 המסמכים הרלוונטיים (טקסט המטלה, L08 summary, Part-A/B/C,
ההנחיות המקצועיות) והבנה מלאה של הדרישות הטכניות (AirLLM/CPU) והסטנדרט התוכנתי.

## שלב 2 — תכנון (Plan Mode)

**החלטות שהתקבלו באמצעות שאלות ממוקדות למשתמשת:**
1. מיקום הפרויקט: תיקיית `HW5` חדשה לצד HW1-HW4.
2. גודל מודל: הוחלט להשאיר לשיקול דעת הסוכן ("אתה תחליט כך שיתאים למה שנדרש כך
   שאקבל ציון גבוה") — נבחר `microsoft/Phi-3-medium-4k-instruct` (~14B, MIT,
   לא-gated) בהתבסס על מפרט החומרה שנבדק בפועל (i7-1165G7, 76GB RAM, ללא GPU).
3. היקף העבודה בשיחה: תכנון + שלד פרויקט מלא, ללא הרצה בפועל.

**תוצאה:** מסמך תוכנית (`quiet-sniffing-rocket.md`) שאושר על ידי המשתמשת, כולל
בחירת מודל מנומקת (ADR), מבנה פרויקט מלא, ורצף ביצוע שלב-אחר-שלב.

## שלב 5 — קריאת מסמך המטלה הרשמי המלא (`ex05-AirLLM.pdf.pdf`) ועדכון מקיף

**פרומפט:** "שכחתי להוריד את המסמך הכי חשוב שמתאר באמת את המטלה — תשתמש בו כולו
ואל תשכח אף פירוט ואף תנאי, וכל שינוי כמובן שינוי בקבצי PRD+PLAN+TODO ופוש
לגיטהאב, ותעשה מה שנדרש באמת פה אבל תגיד לי שלב שלב".

**תוצאה:** קריאת `ex05-AirLLM.pdf.pdf` המלא (14 עמודים) וזיהוי דרישות שלא היו
מתועדות: שאלות מחקר ספציפיות, ניתוח כלכלי חובה (On-Prem מול API + נקודת איזון),
מדידת TTFT/TPOT נפרדת (לא קירוב), README חייב לשמש כדוח הסופי עצמו, אזהרה נגד
מודל "גדול מדי" גם עבור AirLLM, דרישת הרחבה מקורית אחת, ונתיב ייעודי לשכבות
AirLLM. עודכנו: `docs/PRD.md` (v1.01), `docs/PLAN.md` (ADR-4/5/6), `docs/TODO.md`,
נוסף `docs/PRD_economic_analysis.md`; נוסף קוד: `CostAnalysisService`,
`StreamingTimingMixin`, עדכון `RunMetrics`, הרחבת `ReportService` (break-even +
Model Roofline), עדכון בדיקות בהתאם. כל שלב נדחף בנפרד ל-GitHub.
