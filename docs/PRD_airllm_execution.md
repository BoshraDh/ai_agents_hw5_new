# PRD — מנגנון: AirLLM Execution Service

## תיאור מפורט

`AirllmService` מריץ את **אותו** מודל בדיוק (`microsoft/Phi-3-medium-4k-instruct`)
דרך חבילת `airllm`, אשר טוענת את המודל שכבת-טרנספורמר-אחת-בכל-פעם באמצעות `mmap`
על קובצי SafeTensors: מבצעים forward pass עבור השכבה, ואז משחררים אותה מהזיכרון
לפני טעינת השכבה הבאה. זהו יישום ישיר של עקרונות ניהול הזיכרון הווירטואלי
(Virtual Memory, Page Cache, Page Fault) שנלמדו בפרקי ההרצאה (L08 §8, Part-B, Part-C).

### רקע תיאורטי (מההרצאה)

בזמן ש-Baseline דורש שכל 14 מיליארד הפרמטרים יהיו ב-RAM בו-זמנית, AirLLM צריך
להחזיק בזיכרון רק שכבה אחת (מתוך ~40 שכבות בטרנספורמר בגודל הזה) פלוס hidden state
קטן — כלומר **peak RAM נמוך בסדרי גודל**. המחיר: I/O מהדיסק בכל טעינת שכבה (page
fault → קריאה מ-NVMe SSD → page cache) גורם ל-latency גבוה משמעותית לכל טוקן. זהו
בדיוק ה"מסחר" (trade-off) בין זיכרון לזמן שמודגש בהרצאה: "AirLLM הוא גשר בין מודל
ענק לבין זיכרון מוגבל — בעזרת טעינה חלקית ומודעות למחיר ה-I/O".

## דרישות ספציפיות

- קלט/פלט זהים בדיוק ל-`ModelLoaderService` (אותו `RunMetrics`) — כדי לאפשר השוואה
  ישירה שורה-מול-שורה.
- שימוש בפרמטר `compression` של חבילת `airllm` (ללא קוונטיזציה בברירת המחדל, כדי
  לבודד את אפקט "טעינה שכבה-אחר-שכבה" בנפרד מאפקט הקוונטיזציה — זה נבדק בנפרד
  ב-`QuantizationService`).
- תיעוד מפורש בפלט של מספר ה-page-faults/I/O reads המוערך (אם ניתן לחלץ מ-
  `airllm`, אחרת חישוב עקיף: גודל המודל בבייטים חלקי גודל page).

## אילוצים וחלופות שנשקלו

- נשקל להריץ AirLLM גם על מודלים גדולים בהרבה (30B+) כדי להדגים קיצון; נדחה בשלב
  זה מטעמי זמן ריצה (ר' PRD ראשי, ADR-1) — מתועד כהרחבה עתידית אפשרית.
- **אומת בפועל (Phase 4)**: `airllm.AutoModel.from_pretrained` ברירת המחדל שלו היא
  `device="cuda:0"` — ללא ציון מפורש `device="cpu"`, הטעינה נכשלת עם
  `"Torch not compiled with CUDA enabled"` (נצפה בניסיון הראשון על המחשב הזה,
  שאין בו GPU). תוקן ב-`airllm_service.py` — `device="cpu"` מועבר תמיד, כי זו
  עובדה פיזית קבועה על סביבת ההרצה (לא ערך תצורה משתנה), ומתועדת כזיהוי חי
  (לא רק "נשקל") מ-Phase 4.

## קריטריוני הצלחה ותרחישי בדיקה

- **Happy path**: אותה קבוצת פרומפטים כמו ב-Baseline מניבה טקסט (לא בהכרח זהה
  מילה-במילה עקב הבדלי מימוש פנימיים, אך סמנטית קרובה).
- **Edge case — קובץ מודל לא במטמון מקומי**: הורדה ראשונה דרך `ApiGatekeeper` לפני
  הרצת AirLLM (בדיקה עם mock שמוודא שהגייטקיפר נקרא ולא `huggingface_hub` ישירות).
- **Edge case — דיסק איטי/חסימת I/O**: מדומה בבדיקה עם `time.sleep` מבוקר ב-mock,
  ומוודא שה-timeout מקונפיגורציה (`rate_limits.json`) נכבד.
