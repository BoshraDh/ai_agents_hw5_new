# PRD — מנגנון: Benchmark Orchestration & Reporting

## תיאור מפורט

שני שירותים המשלימים זה את זה:

- **`BenchmarkService`**: מריץ קומבינציות שיטתיות של {שיטה × פרומפט × אורך פלט
  מבוקש} על פני שלוש השיטות (Baseline/AirLLM/Quantized), אוסף `RunMetrics` לכל
  ריצה, ושומר הכל כקובץ JSON יחיד תחת `results/run_<timestamp>.json`. זהו מנגנון
  ניתוח הרגישות (Sensitivity Analysis) הנדרש בהנחיות §9.1 — הפרמטר הנסרק הוא אורך
  הפלט המבוקש (`max_new_tokens`), עם שיטת "פרמטר-אחד-בכל-פעם" (OAT).
- **`ReportService`**: קורא קובצי `results/*.json` (ללא תלות בהרצה מחדש של
  המודלים) ומפיק: (א) טבלת השוואה מרוכזת (CSV/Markdown), (ב) bar chart להשוואת
  peak RAM בין שלוש השיטות, (ג) line chart של tokens/sec מול אורך הפלט לכל שיטה,
  (ד) heatmap אופציונלי (שיטה × אורך פלט → latency). כל הגרפים נשמרים תחת `assets/`.

## דרישות ספציפיות

- `BenchmarkService` תלוי אך ורק בממשק המשותף של שלושת שירותי ההרצה (לא יודע דבר
  על הפרטים הפנימיים של כל שיטה) — מאפשר הוספת שיטה רביעית בעתיד ללא שינוי כאן
  (ר' PLAN.md, ADR-2).
- `ReportService` משתמש ב-`matplotlib`/`seaborn` (תלות מוגדרת ב-`pyproject.toml`),
  צבעים עקביים ונגישים, כותרות/צירים ברורים, רזולוציה גבוהה (≥150 dpi) לפי דרישות
  האיכות בהנחיות §9.3.
- קובצי התוצאות הגולמיים (JSON) הם ה"מקור אמת" — ה-notebook וה-`ReportService`
  קוראים מהם בלבד, לא שומרים state כפול.

## אילוצים וחלופות שנשקלו

- נשקל לשמור תוצאות ב-SQLite במקום JSON; נדחה לטובת JSON פשוט — הנפח (עשרות-מאות
  רשומות) לא מצדיק מסד נתונים, ו-JSON קריא וניתן ל-diff ב-git.
- נשקל לייצר את הגרפים בזמן ריצת ה-benchmark עצמו; נדחה — הפרדת "איסוף נתונים"
  מ"ניתוח/הצגה" מאפשרת להריץ מחדש רק את שלב הדוחות בזול אם רוצים לשנות עיצוב גרף.

## קריטריוני הצלחה ותרחישי בדיקה

- **Happy path**: `run_full_benchmark_suite()` מפיק קובץ JSON תקין עם N רשומות
  (N = 3 שיטות × מספר פרומפטים × מספר אורכי-פלט); `generate_report()` מפיק לפחות
  2 קובצי גרף + טבלה אחת.
- **Edge case — ריצה חלקית נכשלה**: אם שיטה אחת נכשלת (למשל Ollama לא זמין),
  שאר הריצות ממשיכות והקובץ הסופי מכיל את הרשומות המוצלחות + רשומות `succeeded=False`
  עבור הכושלות (graceful degradation, לא קריסה כוללת).
- **Edge case — תיקיית results ריקה**: `generate_report` על תיקייה ריקה מחזיר
  שגיאה ברורה במקום גרף ריק/שגוי.
